Node.js 시작하기

Apr 26, 2022 in nodejs, non-blocking

논블로킹

논블로킹: 오래 걸리는 함수를 백그라운드로 보내서 다음 코드가 먼저 실행되게 하고, 나중에 오래 걸리는 함수를 실행

  • 논 블로킹 방식 하에 일부 코드는 백그라운드에서 병렬로 실행됨
    • 일부 코드: I/O 작업, 압축, 암호화 등등
  • 나머지 코드는 블로킹 방식으로 실행됨
  • 그래서 I/O 작업이 많을때 노드 활용성이 극대화됨

프로세스와 스레드

  • 프로세스: 운영체제에서 할당하는 작업의 단위, 프로세스간 자원 공유 X
  • 쓰레드: 프로세스 내에서 실행되는 작업의 단위, 부모 프로세스간 자원 공유 O

노드 프로세스는 멀티 스레드이지만, 직접 다룰 수 있는 스레드는 하나이기 때문에 싱글 스레드라고 표현

싱글 쓰레드

싱글 스레드라 주어진 일을 하나밖에 처리하지 못함

  • 블로킹이 발생하는 경우 나머지 작업은 모두 대기해야함 -> 비효율

대신 노드는 논블로킹 모델을 채택하여 일부 코드를 백그라운드 (다른 프로세스)에서 실행 가능하게 만듦

  • 요청을 먼저 받고, 완료될 때 응답함
  • I/O 관련 코드가 아닌 경우 싱글 스레드, 블로킹 모델과 같아짐

멀티 스레드 모델과의 비교

주방에 비교

  • 점원: 스레드
  • 주문: 요청
  • 서빙: 응답

싱글 스레드 블로킹 모델에서는 점원이 하나의 요청과 응답을 완료한 후, 다음 고객으로 넘어감

싱글 스레드 논블로킹 모델에서는 모든 주문을 받고, 완료될 때 응답함

싱글 스레드의 단점은 에러를 처리하지 못하는 경우에 멈추게 됨. 반대로 멀티 스레드 모델은 에러 발생시 새로운 스레드를 생성하여 극복

하지만 멀티 스레드로 프로그래밍하기에 난이도가 높고, 그를 피해서 싱글 스레드로 처리하려고 노드로 개발함
